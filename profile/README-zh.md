# ESCSS - 逃離開發者的噩夢
![logo](https://github.com/ESCSS-labs/ESCSS/blob/main/assets/logo.png)

## 緣起
受到《人月神話》一書的啟發，常常思考著這個問題：過去 50 年所尋求的“銀彈”真的存在，還是僅僅是一種幻想？

回顧我的過往，發現自己常常有一些奇特的想法。隨著這些概念逐步的驗證，我認為銀彈是似乎是可行的，所以花了一段時間，自主投入研究。

經過一段時間的反覆沈思與實驗，這就是我的成果 ESCSS —— 專注在提升開發者體驗與效率，希望讓軟體產業更好，同時也從我的工作中獲得報酬。

## 特色
  - **多功能性**: 靈活的 JavaScript 和 CSS 工具，與主流的網路技術兼容。
  - **安全性**: 開放核心模型確保了強大的透明度，並且 ESTest 沒有依賴項，安全風險降至最低。
  - **非侵入性**: Legacy code，無縫使用。
  - **開發者體驗**: 開發者做事順手，產品品質上升。
  - **100% 輸入覆蓋率**: 全面的輸入測試覆蓋率可以最大限度地減少錯誤，增強客戶信任。
  - **錯誤捕獲**: 常常通靈，花費數小時，找尋錯誤？數秒內定位錯誤來源。
  - **CSS 效率提升**: 通過 CSS 工具，優雅的提升開發效率以及開發者的滿意度。

## 這兩種許可證有什麼不同？
主要區別在於 您的項目是否**保持開源或保持私有**。這兩種授權都提供**相同的功能—沒有額外的商業組件，現在和將來都是如此**。

我選擇“開放核心”商業模式，因為我認為這種方式可以協助軟體生態更好，同時我從我的工作中獲得收入。
